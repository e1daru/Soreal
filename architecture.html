Soreal — System Workflow

Knowledge-Graph RAG Pipeline for Legal Document Analysis

👤
User
Uploads a legal document or asks a question via the web interface
Browser SPA
🌐
FastAPI Server
Receives requests, streams progress via SSE, returns JSON answers
Uvicorn · Port 8001
Router
LangGraph Entry Node
📄 Analyze Document
1. Chunk & Embed
Split document into overlapping chunks, embed with HuggingFace
bge-small-en-v1.5 · 384d
2. Extract Entities
Legal NER via LLM — finds parties, clauses, obligations, dates, fees…
Ollama · llama3.1:8b
3. Form Triplets
Extract (subject → predicate → object) relations from 30 legal predicates
Ollama · llama3.1:8b
4. Build Knowledge Graph
Create entity nodes + relation edges in SurrealDB with vector indexes
SurrealDB · 18 node · 30 edge tables
❓ Ask Question
1. Understand Query
LLM decides which tools to call — vector search, graph summary, or mutation
Ollama · tool-calling
2. Search Knowledge Graph
Vector similarity search across 11 tables + graph traversal for related edges
HNSW · cosine · top-K
3. Generate Answer
LLM synthesizes answer from retrieved graph facts, citing source chunks
Ollama · llama3.1:8b
🔄 loops until answer is complete
4. Score Groundedness
Token overlap + semantic similarity between answer and graph facts
0.6 × coverage + 0.4 × cosine sim
✅
Response to User
Analyze: entity count, triplet count, graph stats
Ask: answer + groundedness score + source citations
